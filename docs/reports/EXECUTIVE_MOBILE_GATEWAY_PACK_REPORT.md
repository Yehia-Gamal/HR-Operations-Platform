# Executive Mobile + Secure Admin Gateway Pack

تمت إضافة حزمة تشغيل جديدة للسيستم تركّز على دخول الإدارة بشكل غير ظاهر للموظفين، وصفحة متابعة تنفيذية موبايل للشيخ محمد، وطلب الموقع المباشر بموافقة الموظف.

## ما تم تنفيذه

### بوابة تشغيل الإدارة
- إضافة رابط جديد: `/operations-gate/` باسم **بوابة التشغيل**.
- الرابط العام للنظام يظل مخصصًا لتطبيق الموظفين فقط.
- `admin-login.html` أصبح يحوّل إلى بوابة التشغيل بدل فتح لوحة الإدارة مباشرة.
- في النشر الحقيقي، الدخول المباشر إلى `/admin/` يتم تحويله إلى بوابة التشغيل ما لم تكن الجلسة قد فتحت البوابة.
- هذه البوابة ليست بديلًا عن الصلاحيات؛ الحماية الأساسية ما زالت من تسجيل الدخول والأدوار.

### صفحة تنفيذية موبايل للشيخ محمد
- إضافة رابط مباشر: `/executive/`.
- إضافة صفحة داخل لوحة الإدارة: `#executive-mobile`.
- تعرض الصفحة:
  - إجمالي الموظفين.
  - الحاضرين اليوم.
  - المتأخرين.
  - الغائبين.
  - من لديهم إجازة.
  - طلبات الموقع المعلقة.
- تعرض كروت الموظفين بالصور.
- عند النقر على صورة/تفاصيل الموظف تظهر صفحة تفاصيل اليوم.

### تفاصيل الموظف التنفيذية
- حالة اليوم.
- وقت الحضور.
- وقت الانصراف.
- آخر موقع محفوظ أو مرسل.
- آخر 7 أيام حضور.
- آخر الإجازات والمأموريات.
- سجل طلبات الموقع المباشر.
- رابط فتح الموقع على Google Maps.

### طلب الموقع المباشر
- من صفحة الشيخ محمد يمكن طلب موقع أي موظف.
- الموظف يستقبل الطلب داخل صفحة **الموقع** في تطبيق الموظفين.
- الموظف يختار:
  - إرسال موقعي.
  - رفض مع كتابة السبب.
- عند الإرسال يتم قراءة GPS بدقة عالية وحفظ الرد.
- عند الرفض يتم حفظ سبب الرفض.
- يتم تسجيل العملية داخل سجل التدقيق.

### سجلات دخول الإدارة
- إضافة endpoint محلي وسجل Supabase لمحاولات دخول الإدارة.
- تسجيل نجاح وفشل دخول الإدارة من الواجهة.

## ملاحظات مهمة

- كود بوابة التشغيل الافتراضي للتجربة المحلية هو `LOCAL-DEMO-GATE` ويجب تغييره قبل النشر.
- لا يوجد تتبع مستمر أو خفي للموظف. النظام يطلب الموقع مرة واحدة، والموظف يوافق أو يرفض.
- تشغيل الميزة على Supabase يتطلب تطبيق Patch رقم `023` بعد كل الـ patches السابقة.
