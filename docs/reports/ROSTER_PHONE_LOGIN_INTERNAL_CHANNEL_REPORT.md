# تقرير تطبيق تعديلات قائمة الموظفين والقناة الداخلية — v1.3.3

تم تطبيق التعديلات المطلوبة على النسخة:

- حذف التسجيل الذاتي من تطبيق الموظف.
- جعل إنشاء حسابات الموظفين من لوحة HR فقط.
- اعتماد قائمة الموظفين الموجودة في `بيانات الموظفين.xlsx` كقائمة رسمية بعدد 28 موظف.
- ربط كل موظف بمديره المباشر، رقم هاتفه، مسماه الوظيفي، والصورة المتاحة من ملف الصور.
- عدد الصور التي تم ربطها فعليًا: 20.
- تفعيل دخول الموظفين برقم الهاتف، وكلمة المرور الافتراضية تساوي الرقم الشخصي/رقم الهاتف المسجل.
- إضافة لجنة حل المشاكل والخلافات إلى واجهة المدير التنفيذي.
- تقوية قناة التواصل الداخلي لإرسال الإعلانات والتذكيرات والتعليمات مع Web Push وتنبيه صوتي داخل التطبيق عند فتحه.
- فرض بصمة الجهاز/Passkey قبل تسجيل الحضور والانصراف وقبل إرسال الموقع.

## ملفات مهمة

- `shared/data/authorized-employees.json`
- `docs/templates/authorized-employees-import.json`
- `supabase/sql/patches/042_authorized_roster_phone_login_internal_channel.sql`
- `supabase/migrations/20260502074007_042_authorized_roster_phone_login_internal_channel.sql`

## ملاحظة أمنية

استخدام رقم الهاتف ككلمة مرور أضعف من كلمات المرور العشوائية. تم تطبيقه حسب الطلب، لكن يفضل لاحقًا إضافة OTP أو إجبار تغيير كلمة المرور بعد أول دخول.

## ربط الصور بدون شحنها داخل ملفات الواجهة

تم نقل صور الموظفين المطابقة إلى مسار استيراد غير مستخدم مباشرة من الواجهة:

`supabase/storage-import/avatars/employee-avatars/`

والنظام يستخدم مسارات تخزين من نوع:

`employee-avatars/<employee-id>.png`

عند رفع هذه الصور إلى Supabase Storage داخل bucket باسم `avatars` ستظهر تلقائيًا في لوحة الإدارة وتطبيق الموظف والمدير التنفيذي، لأن الواجهة أصبحت تحول مسار التخزين إلى رابط عام من إعدادات Supabase.

## ملاحظات مهمة بعد النشر

- الموظفون الذين لم يتم العثور على صورة مطابقة لهم سيظهرون بأحرف الاسم تلقائيًا.
- إنشاء Auth Users في Supabase لا يتم من SQL patch فقط؛ يتم من لوحة HR أو Edge Function `admin-create-user` بعد تشغيل patch 042.
- البصمة/Passkey تحتاج HTTPS أو localhost وجهاز يدعم WebAuthn.
