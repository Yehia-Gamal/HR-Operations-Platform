# تقرير حزمة الاستكمال والتقوية — Continuity Hardening Pack

تاريخ الحزمة: 2026-04-29

## الهدف
استكمال إضافات الإدارة والموظفين بعد حزمة بوابة التشغيل والمتابعة التنفيذية، مع التركيز على منع الأخطاء الخطيرة، تحسين تجربة الرد على طلب الموقع، وتقوية بوابة الدخول.

## ما تم تنفيذه

### 1. مركز الاعتمادات الحساسة
تمت إضافة صفحة جديدة داخل لوحة الإدارة:

- اعتمادات حساسة
- تعرض الطلبات الحساسة المعلقة والمنفذة والمرفوضة.
- يمكن للمدير التنفيذي أو التقني صاحب الصلاحية اعتماد وتنفيذ الطلب أو رفضه.
- أي إجراء حساس يمكن ربطه بمسار اعتماد قبل التنفيذ.

### 2. حماية حذف الموظفين
في الوضع المحلي:

- إذا حاول مستخدم غير تنفيذي حذف موظف، لا يتم الحذف مباشرة.
- يتم إنشاء طلب اعتماد حساس يظهر في صفحة الاعتمادات.
- عند الاعتماد يتم تنفيذ الحذف المنطقي وتعطيل حساب المستخدم المرتبط.
- يتم تسجيل العملية في Audit Log.

### 3. تقوية بوابة التشغيل
تم تطوير `/operations-gate/`:

- قفل مؤقت بعد 5 محاولات خاطئة.
- مدة القفل 15 دقيقة.
- تسجيل وقت قبول البوابة داخل sessionStorage.
- توضيح أن كود البوابة ليس بديلًا عن الصلاحيات.

### 4. تحسين تطبيق الموظف في طلب الموقع
تم تحسين الصفحة الرئيسية للموظف:

- يظهر عداد طلبات الموقع المباشر المعلقة.
- إذا طلبت الإدارة الموقع، يظهر تنبيه عاجل في الرئيسية.
- زر مباشر للانتقال إلى صفحة الموقع والرد على الطلب.

### 5. تجهيز Supabase Patch جديد
تمت إضافة:

```text
supabase/sql/patches/024_sensitive_approvals_gateway_hardening.sql
```

ويضيف:

- `sensitive_approvals`
- `executive_presence_snapshots`
- سياسات RLS مناسبة للمدير التنفيذي والإدارة.
- صلاحيات جديدة:
  - `sensitive-actions:approve`
  - `sensitive-actions:request`
  - `executive:presence-map`

### 6. تحديث إصدار التخزين والكاش
تم رفع نسخة التخزين المحلي إلى:

```text
hr-attendance.local-db.v13
```

وتم تحديث إصدار ملفات الواجهة إلى:

```text
mobile-ui-push-fix-20260430-02
```


## ملاحظات إنتاجية

في Supabase الحقيقي، اعتماد العمليات الحساسة يجب أن يكتمل عبر Edge Function أو RPC آمن عند الحاجة لتنفيذ عمليات خطيرة مثل حذف موظف أو تعطيل مستخدم، حتى لا يتم تنفيذها من المتصفح مباشرة.
