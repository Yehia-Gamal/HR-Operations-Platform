# تقرير Stability Pack

تم تنفيذ حزمة إصلاحات على الحفظ المحلي، إنشاء الموظفين، الإجازات، الشكاوى، وخزنة كلمات المرور المؤقتة.

## ما تم إصلاحه

- رفع إصدار التخزين المحلي إلى `v9` مع قراءة بيانات `v8` تلقائيًا.
- إضافة رسالة واضحة عند امتلاء مساحة التخزين المحلي بدل فشل صامت.
- ضغط صور الموظفين قبل حفظها داخل LocalStorage لتقليل مشاكل عدم الحفظ بعد إعادة تحميل الصفحة.
- جعل البريد الإلكتروني مطلوبًا عند إنشاء موظف مرتبط بحساب دخول.
- عند إضافة موظف جديد يتم إنشاء كلمة مرور قوية تلقائيًا إذا تركت الحقل فارغًا.
- إضافة خزنة كلمات مرور مؤقتة للتقني/الإدارة العليا مع إمكانية إصدار كلمة جديدة.
- إصلاح إرسال طلب الإجازة من تطبيق الموظف والربط الصحيح بالموظف الحالي.
- إصلاح Payload الإجازات والمأموريات في Supabase حتى يستخدم أسماء الأعمدة الصحيحة.
- تطبيق لجنة حل المشاكل والخلافات: أبو عمار، ياسر فتحي، بلال الشاكر، يحيى جمال، الشيخ محمد يوسف.
- عند إنشاء شكوى يتم إخطار كل أعضاء اللجنة، وعند التصعيد يتم إخطار يحيى والشيخ محمد.
- توضيح شريط الوضع المحلي حتى يعرف المستخدم أن الحفظ المحلي خاص بالمتصفح فقط.

## ملاحظات مهمة

- في الوضع المحلي يتم الحفظ داخل متصفح نفس الجهاز، وليس داخل ملفات المشروع.
- للحفظ الحقيقي بين كل الأجهزة يجب تفعيل Supabase في `shared/js/supabase-config.js` وتشغيل SQL patches ونشر Edge Functions.
- في الإنتاج لا يمكن ولا يجب عرض كلمات المرور الأصلية؛ المتاح أمنيًا هو إصدار كلمة مرور مؤقتة جديدة.
