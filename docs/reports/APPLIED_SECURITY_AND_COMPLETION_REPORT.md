# تقرير التعديلات المطبقة على نسخة المشروع

## أمان وتشغيل

- تنظيف ملف استيراد الموظفين من بيانات حقيقية وكلمات مرور متوقعة.
- إزالة الاعتماد على كلمات مرور مبنية على آخر أرقام الهاتف.
- إضافة `phone` إلى `profiles` في الـ schema الأساسي.
- إصلاح `resolve_login_identifier` لعدم الاعتماد على `profiles.is_active` غير الموجود.
- إضافة جدول rate-limit لمحاولات حل رقم الهاتف إلى بريد.
- تحديث Edge Function الخاصة بتسجيل الدخول بالهاتف لاستخدام hashing للـ IP والرقم مع حد محاولات.
- جعل إعدادات Supabase الافتراضية آمنة: `enabled=false` و `strict=true` حتى يتم ضبط المفاتيح الحقيقية.
- إضافة أعمدة `bucket_id` و `storage_path` للمرفقات.
- تحويل روابط المرفقات الخاصة إلى Signed URLs مؤقتة بدل Public URLs.
- تقوية سياسة تغيير كلمة المرور داخل الواجهة.
- حذف `.git` و `supabase/.temp` من نسخة التسليم المضغوطة.

## PWA وتشغيل

- إضافة صفحة الإدارة وملفاتها إلى Service Worker حتى لا يفشل fallback الخاص بالإدارة.
- تحديث cache version لمنع تعارض الكاش القديم.
- إضافة قائمة Production Checklist.

## حدود الفحص

- لا يمكن اختبار الاتصال الحقيقي بقاعدة Supabase من هذه البيئة لأن مفاتيح المشروع الحقيقية غير موجودة.
- تم اختبار البنية والملفات محليًا، وفحص Syntax لملفات JavaScript الأساسية.
