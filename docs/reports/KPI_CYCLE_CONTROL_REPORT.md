# تقرير استكمال KPI — Cycle Control v1.2.2

الإصدار: `1.3.1-runtime-alignment`  
آخر Patch مطلوب: `040_runtime_alignment_fix.sql`

## ما تم إضافته

1. **نافذة تقييم KPI رسمية**
   - يبدأ التقييم يوم 20 من كل شهر.
   - ينتهي تسليم الموظف/المدير يوم 25 من نفس الشهر.
   - بعد يوم 25 تكون المراجعة المتأخرة مخصصة لـ HR والسكرتير التنفيذي والمدير التنفيذي.
   - بعد الإغلاق لا يتم التعامل إلا من صاحب الصلاحية التنفيذية أو الكاملة.

2. **متابعة مراحل الاعتماد**
   - لم يبدأوا.
   - بانتظار المدير.
   - بانتظار HR.
   - بانتظار السكرتير التنفيذي.
   - بانتظار المدير التنفيذي.
   - اعتماد نهائي.

3. **أزرار إدارة دورة KPI**
   - تجهيز التقييمات الناقصة.
   - إرسال تذكيرات KPI حسب المرحلة.
   - إغلاق دورة KPI.

4. **تطبيق الموظف**
   - يعرض حالة نافذة التقييم.
   - يمنع إرسال التقييم الذاتي خارج نافذة 20–25.
   - يوضح أن بنود HR فقط لا يمكن للموظف تعديلها.

5. **قاعدة البيانات**
   - إضافة Patch رقم 038.
   - إضافة View لمتابعة الدورة: `kpi_cycle_stage_report`.
   - إضافة View لمتابعة كل موظف: `kpi_employee_stage_report`.
   - إضافة أعمدة إغلاق/اعتماد للدورة والتقييمات.

## ملاحظات إنتاجية

عند النشر الحقيقي شغّل SQL حتى Patch 038، ثم اختبر دورة KPI بحسابات فعلية لكل من: موظف، مدير مباشر، HR، سكرتير تنفيذي/تقني، ومدير تنفيذي.
